Placeholder.js クイックスタートガイドHTMLを書き換えずに、外部のJSONやCSVからデータを自動注入するための軽量スクリプト「Placeholder.js」の使い道まとめだよ！1. 基本の書き方（JSON）HTML要素に特定の「データ属性」を付与するだけでOK！<!-- テキストを書き換える場合 -->
<h1 data-source="json" 
    data-file="./files/data.json" 
    data-key="project.name">
    読み込み中...
</h1>

<!-- 属性（リンク先や画像）を書き換える場合 -->
<img src="" 
     data-source="json" 
     data-file="./files/data.json" 
     data-key="author.avatar" 
     data-attr="src">
2. 設定できる属性（Data Attributes）属性名内容備考data-sourceデータの形式json または csvdata-fileファイルのURLローカルパスも外部URL（API）もOK！data-key取得したいキードット記法（user.profile.name）対応data-attr書き換える対象src, href, value など（未指定ならテキスト）data-errorエラー時の挙動log を指定すると画面にエラーを表示data-idCSV用の行特定IDCSV利用時に、どの行を出すか決めるID3. データの準備（JSONの例）data-key で指定する構造に合わせて作成してね。{
  "project": {
    "name": "Dynamic App",
    "version": "1.0.0"
  },
  "author": {
    "avatar": "[https://example.com/img.png](https://example.com/img.png)"
  }
}
4. 外部API（GitHubなど）を使う時のコツdata-file に直接APIのURLを書き込めば、リアルタイムデータを表示できるよ！例: https://api.github.com/repos/ユーザー名/リポジトリ名注意: 外部サーバーがCORS（アクセス許可）を出している必要があるよ。（GitHub APIはOK！）5. Placeholder.js の読み込みHTMLの最後（</body>の直前）で読み込ませてね。<script src="Placeholder.js"></script>
覚えておくと便利なポイントキャッシュ機能: 同じファイルを何度も読み込まないようにスクリプト内でキャッシュしてるから、たくさんの場所で使っても動作が軽いよ！エラー処理: data-error="log" を入れておけば、ファイルがなかった時に赤文字で教えてくれるからデバッグが楽だよ。これがあれば、Cのプロジェクトがどんどん動的でカッコよくなっていくね！応援してるよ！
